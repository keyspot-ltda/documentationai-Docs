## Antes de entrar em contato

Para agilizar o atendimento, inclua as seguintes informações no seu chamado:

<Steps>
  <Step title="Identifique o problema" icon="search">
    Descreva o que você está tentando fazer e o que está acontecendo. Inclua:

    - Endpoint chamado (ex: `GET /v1/properties`)
    - Parâmetros enviados
    - Código HTTP retornado
    - Body da resposta de erro (com `error` e `code`)
  </Step>
  <Step title="Inclua contexto técnico" icon="code">
    - Linguagem/framework utilizado
    - Timestamp aproximado do erro (UTC)
    - Sua `API Key` (nunca envie o `API Secret`)
    - IP de origem das requisições
  </Step>
  <Step title="Verifique a documentação" icon="book-open">
    Consulte os seguintes recursos antes de abrir um chamado:

    - [Erros Comuns](/operacao/erros-comuns) — diagnóstico por código de erro
    - [Boas Práticas](/guias/boas-praticas) — paginação, retry e rate limiting
    - [Autenticação](/comece-aqui/autenticacao) — fluxo JWT e renovação de token
  </Step>
</Steps>

## Problemas comuns

| Sintoma | Provável causa | Solução rápida |
|---------|---------------|----------------|
| `401 INVALID_TOKEN` em todas as chamadas | Token expirado | Gere um novo token |
| `403 IP_BLOCKED` | IP não autorizado | Solicite inclusão do IP |
| `429` frequente | Muitas requisições | Reduza a frequência ou solicite aumento de limite |
| Dados desatualizados | Cache local | Use `updatedAfter` para sincronização incremental |
| Campos vazios na resposta | Campo não solicitado | Adicione o campo ao parâmetro `fields` |
